Always terminate dependent async-s with reason 'shutdown'.

If we're explicitly terminating dependent async-s due to some kind of
error, it means that this error will be visible either because it's
propagated to the user of the async or because a crash report will be
generated by our async. Since the error is already visible, there's no
point in propagating it further down the async tree.

[531613] Add "statemachine" new grammar and meta-model based on XText

* There 3 new plug-ins :
** org.eclipse.sirius.tests.sample.xtext.statemachine
** org.eclipse.sirius.tests.sample.xtext.statemachine.ide
** org.eclipse.sirius.tests.sample.xtext.statemachine.ui
That plug-ins have been created from "New XText Project" command.
The grammar file
/org.eclipse.sirius.tests.sample.xtext.statemachine/src/org/eclipse/sirius/tests/sample/xtext/Statemachine.xtext
is the only file that has been changed manually. The whole rest of the
file has been generated by XText tooling.
The code generation has to be run after any change in not src-gen folder
including xtend file which corresponding java file is generated.

* org.eclipse.sirius.tests.sample.xtext.statemachine plug-in contains
the statemachine XText grammar and the generated EMF meta-model

* org.eclipse.sirius.tests.sample.xtext.statemachine.ide and
org.eclipse.sirius.tests.sample.xtext.statemachine.ui plug-ins contain
all necessary to have the xtext editor dedicated to statemachine file
extension, with all necessary commands, auto-completion, validation
etc...

DESCRIPTION:

CBQE-474 modify LoadDocumentTask to make KVStore store either the hash of item's value or item's value itself

Right now kvstore directly store the item's value generated by BlobGenerator or other load generator tasks.
If the item size is big or you want to load several millions of items during your automation,
this code design will consume a lot of memory on the client site. I parametize the choice for kvstore to store
either the item' hash value or item's value in LoadDocumentsTask.

InMemory/Persisted Stability Timestamps

Allow Stability Timestamps to be either InMemory or Persisted.
The frequency of both the timestamps can be controlled using
- IN_MEM_TS_TRIGGER
- PERSISTED_TS_TRIGGER

Based on the flag set by timekeeper, storage manager chooses
the forestdb snapshot option.

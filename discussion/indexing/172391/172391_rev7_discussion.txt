DESCRIPTION:

MB-51780: Set MemDB iterator Refresh rate during Scan

MemDb NewIterator is not initialized with the refresh rate
from the config. There is SetRefreshRate API but is not set
prior to scan. If refresh rate remains 0, we do not release
barrier session.  This can impact memory reclaim with long
running scans with delete heavy workload.

Depending on workload, reclaim activity during scan can add
some overhead in scan throughput & latency. But early reclaim
should help lower chances of pauses.
perf tests:
https://docs.google.com/document/d/1JtbnnqAo7-K9lbdk8BVE-zdDqnlHU-dZzFtfOBV8qp8/edit?usp=sharing

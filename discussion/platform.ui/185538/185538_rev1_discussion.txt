Bug 538944 - Align workspace selection dialog with Microsoft design
guideline

According to
https://docs.microsoft.com/en-us/windows/uwp/design/controls-and-patterns/dialogs-and-flyouts/dialogs
a dialog should:

The dialog title is the main instruction and is optional.

    Use a short title to explain what people need to do with the dialog.
    If you're using the dialog to deliver a simple message, error or
question, you can optionally omit the title. Rely on the content text to
deliver that core information.
    Make sure that the title relates directly to the button choices.
